# 个人技术博客知识库 - 需求文档

## 概述
基于 VitePress 构建一个个人技术知识库博客网站，用于记录和查阅技术积累文档。纯前端实现，无后端依赖，支持 Markdown 驱动内容管理，UI 简洁美观，对 AI 生成内容友好。

## 用户故事

### 1. 文章浏览与导航
1.1 作为用户，我希望看到一个简洁美观的首页，展示最新的技术文章列表，以便快速了解最近的内容更新。
  - 验收标准：
    - 首页展示文章列表，包含标题、摘要、日期、标签
    - 文章按发布日期倒序排列
    - 支持分页浏览（每页 10 篇）
    - 页面加载时间 < 2 秒

1.2 作为用户，我希望能通过分类和标签筛选文章，以便快速找到特定主题的内容。
  - 验收标准：
    - 支持按分类（category）浏览文章
    - 支持按标签（tag）筛选文章
    - 分类和标签页面展示对应文章列表
    - 侧边栏或导航栏展示分类目录

1.3 作为用户，我希望能搜索文章内容，以便快速定位到需要的知识点。
  - 验收标准：
    - 支持全文搜索功能
    - 搜索结果高亮匹配关键词
    - 使用 VitePress 内置的本地搜索（基于 minisearch）

### 2. 文章内容展示
2.1 作为用户，我希望文章页面排版清晰、代码高亮美观，以便舒适地阅读技术文档。
  - 验收标准：
    - 支持 Markdown 完整语法渲染
    - 代码块支持语法高亮和行号显示
    - 支持代码块复制按钮
    - 文章页面有目录（TOC）导航
    - 支持图片展示

2.2 作为用户，我希望网站支持暗色/亮色模式切换，以便在不同环境下舒适阅读。
  - 验收标准：
    - 支持亮色和暗色两种主题
    - 支持手动切换主题
    - 记住用户的主题偏好（localStorage）

### 3. 内容管理（AI 友好）
3.1 作为内容创作者，我希望通过简单地添加 Markdown 文件就能发布新文章，以便 AI 工具能直接生成内容。
  - 验收标准：
    - 在指定目录下新增 `.md` 文件即可自动识别为新文章
    - 使用 frontmatter 定义文章元数据（标题、日期、分类、标签、摘要）
    - 提供标准的 Markdown 文章模板
    - 文章目录结构清晰，按分类组织

3.2 作为内容创作者，我希望有一个标准化的文章模板，以便 AI 生成内容时有统一的格式。
  - 验收标准：
    - 提供 frontmatter 模板，包含所有必要字段
    - 提供文章内容结构模板（问题描述、解决方案、代码示例、参考链接）
    - 模板文件放在项目根目录便于 AI 工具引用

### 4. UI/UX 设计
4.1 作为用户，我希望网站整体风格简洁现代，以便专注于内容本身。
  - 验收标准：
    - 使用 VitePress 默认主题并进行自定义美化
    - 响应式设计，支持移动端浏览
    - 导航栏包含：首页、分类、标签、关于
    - 页脚包含版权信息

4.2 作为用户，我希望首页有个性化的展示，而不是纯文档风格。
  - 验收标准：
    - 首页采用博客风格布局（文章列表 + 侧边栏）
    - 侧边栏展示分类列表和标签云
    - 整体配色协调，字体清晰易读

### 5. 部署与维护
5.1 作为开发者，我希望项目能轻松部署到 GitHub Pages，以便免费托管。
  - 验收标准：
    - 配置 GitHub Actions 自动构建部署
    - 提供部署文档说明
    - 构建产物为纯静态文件

## 技术约束
- 框架：VitePress（最新稳定版）
- 语言：TypeScript / Vue 3
- 样式：CSS 自定义 + VitePress 默认主题扩展
- 搜索：VitePress 内置本地搜索
- 部署：静态文件，支持 GitHub Pages
- 无后端依赖，所有数据基于 Markdown 文件
